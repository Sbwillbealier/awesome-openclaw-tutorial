# OpenClaw v2026.3.13 更新解读（2026-03-14）

> 本文聚焦 OpenClaw GitHub Releases 最新版本 **v2026.3.13** 的新增与修复；适合用来补充到教程的“版本更新”栏目。

##版本概览

- **版本**：v2026.3.13
- **发布日期**：2026-03-14
- **定位**：移动端体验 + 浏览器接管能力增强 + 若干关键修复

##重点更新

###1) 移动端体验升级
- **Android**：聊天设置页重新设计（设备/媒体分组），输入区与会话头部更紧凑，整体更适合移动端高频操作。
- **iOS**：首次启动 onboarding 优化：新增欢迎页；连接步骤更清晰；不再自动打开扫码；Connect 步骤提示使用 `/pair qr`。

###2) 浏览器能力增强（强烈建议写进教程）
- **Chrome DevTools MCP attach**：支持接管“已登录的真实 Chrome 会话”（适合做需要登录态的网站自动化）。
- **内置浏览器 profile**：新增 `profile="user"`（宿主机已登录浏览器）与 `profile="chrome-relay"`（扩展 Relay）。
- **browser.act 自动化增强**：支持批量 actions、selector 定位、延迟点击等，提升稳定性与效率。

###3) Docker：时区可控
- 新增环境变量 `OPENCLAW_TZ`，docker-setup.sh 可固定 gateway/CLI 容器使用的 IANA 时区，避免因宿主机/容器时区不一致导致的定时任务、日志时间混乱。

##关键修复
- **Dashboard v2**：修复工具结果回传导致的聊天历史反复重载/卡顿（工具密集型任务更流畅）。
- **Gateway RPC**：未响应请求有超时回收，避免 pending 状态泄漏。
- **plugin-sdk**：打包策略调整，缓解内存膨胀问题。
- **Ollama**：不再把 reasoning/thinking 字段提升为最终回复文本（避免本地推理泄露到输出）。
- **Control UI（明文HTTP）**：显式 token/password 不再在首次握手前丢失。
